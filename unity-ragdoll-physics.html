学习顺序：先理解“动画控制”和“物理控制”的切换，再看每个骨骼上的 Rigidbody、Collider、Character Joint。

Goal

用一个页面搞懂 Unity 的 Ragdoll Physics

Ragdoll 的本质不是“播放一个死亡动画”，而是让角色的骨骼从 Animator 控制切换成物理系统控制。每个关键骨骼都有自己的 Collider、Rigidbody 和 Joint，死亡时它们会受重力、碰撞和外力影响。

Unity 版本参考：2022.3 LTS

Try It

操作一下

先保持动画控制，再切到 ragdoll。你会看到骨骼不再被站立姿势锁住，而是受重力和约束影响。

关节太松会像绳子，太硬可能抖动。真实项目里还要看质量、碰撞体大小和 solver iterations。

Mental Model

先建立三个核心概念

1. Animator 是正常活着时的控制者

跑、走、攻击、受击通常由 Animator 改骨骼 Transform。这个阶段 ragdoll 的刚体一般设为 kinematic。

2. Rigidbody + Collider 让骨骼进入物理世界

每段主要骨骼有一个简化碰撞体，常见是 Capsule、Box、Sphere。碰撞体不需要完全贴合网格，但要足够合理。

3. Character Joint 限制骨骼不要散架

Joint 把前臂连到上臂、上臂连到肩、头连到脖子，并限制每个方向能转多少。

准备骨骼角色

角色需要是带骨骼的 Skinned Mesh，普通静态模型不能直接像人体一样 ragdoll。

生成物理组件

用 Ragdoll Wizard 给主要骨骼创建 Collider、Rigidbody 和 Character Joint。

修正碰撞体

手臂、腿、头的 collider 太大会悬空，太小会穿地。通常宁可略小也不要明显太大。

修正关节轴

检查 Axis、Swing Axis 和 twist/swing limits，尤其是肘、膝、头、脖子。

死亡时切换

关闭 Animator、NavMeshAgent、主 CapsuleCollider，打开 ragdoll 刚体和碰撞。

Setup

用 Ragdoll Wizard 创建第一版

Unity 的 Ragdoll Wizard 会自动给你生成 Collider、Rigidbody 和 Joint，但自动结果只是起点，后面通常需要手动调整。

操作流程

  • 1把角色实例拖到场景 Hierarchy 里，展开骨骼层级。
  • 2打开 GameObject > 3D Object > Ragdoll...。
  • 3把骨骼 Transform 拖到对应槽位，例如 Pelvis、Left Hips、Left Knee、Left Arm、Head。
  • 4Total Mass 可以先用默认值，例如 20。点击 Create。
  • 5进入 Scene 视图检查 collider 大小和关节方向。
Wizard 槽位 通常对应的骨骼 检查重点
Pelvis Hips / Pelvis 身体重心，很多 ragdoll 的稳定性从这里开始。
Left Hips / Right Hips Upper Leg / Thigh 大腿 collider 不要太粗，否则倒地时像悬空。
Left Knee / Right Knee Lower Leg / Shin 膝盖应主要向一个方向弯，反向限制要小。
Left Arm / Right Arm Upper Arm 肩部和上臂连接点要在肩附近。
Left Elbow / Right Elbow Forearm 肘部 twist axis 经常需要手动修正。
Middle Spine Spine / Spine1 Box Collider 要包住胸腹，不要挤进头或腿。
Head Head 头部 sphere collider 半径要接近头，不只是脸。

Character Joint

参数怎么理解

Character Joint 是 ragdoll 的关键。它像一个可限制角度的球窝关节：物体还是 Rigidbody，但旋转范围被 Low/High Twist 和 Swing Limit 限住。

肘关节例子

前臂能向胸口弯很多，但不能明显向后折。这个“不对称限制”就是 Character Joint 很适合 ragdoll 的原因。

参数 作用 调法
Connected Body 当前骨骼连接到哪个 Rigidbody。为空时连接到世界空间。 前臂连上臂，大腿连骨盆，头连脖子或上脊柱。
Anchor 关节在当前物体本地空间的旋转点。 肘关节放在肘部，膝关节放在膝盖，不要放在骨骼中心。
Axis 主扭转轴，Scene 里通常是橙色 gizmo。 决定 Low/High Twist 的方向。方向错了，限制会像“装反”。
Swing Axis 摆动轴，另一个轴由 Unity 根据前两个轴推导。 用于限制侧向摆动，通常不需要比 twist 调得更细。
Low Twist Limit 主轴负方向限制，单位是度。 肘和膝通常设置成小负值，例如 -5 到 -15，避免反折。
High Twist Limit 主轴正方向限制，单位是度。 肘向前弯可较大，例如 70 到 120，按模型效果调。
Swing 1 / Swing 2 Limit 另外两个方向的对称旋转限制。 角度太小容易不稳定；不用小角度时可以直接锁到 0。
Enable Collision 连接的两个 Rigidbody 是否互相碰撞。 相邻骨骼通常关闭，否则 collider 轻微重叠会互相弹开。
Enable Projection 关节超出限制时强行拉回。 出现拉伸、散架时再开；它能稳住，但可能有突然修正感。
Mass Scale 只影响 joint 求解时的有效质量。 连接的刚体质量差太多会抖，尽量让质量接近。

Runtime Switch

死亡时怎么从动画切到 ragdoll

关键是让两个系统不要同时控制骨骼。活着时 Animator 控制，ragdoll 刚体设为 kinematic；死亡时关闭 Animator 和导航，打开 ragdoll 物理。

RagdollController.cs 核心逻辑示例 
using UnityEngine;
using UnityEngine.AI;

public class RagdollController : MonoBehaviour
{
    [SerializeField] private Animator animator;
    [SerializeField] private NavMeshAgent navAgent;
    [SerializeField] private Collider standingCollider;
    [SerializeField] private Rigidbody[] ragdollBodies;
    [SerializeField] private Collider[] ragdollColliders;

    private void Awake()
    {
        SetRagdoll(false);
    }

    public void Die(Vector3 hitPoint, Vector3 hitForce)
    {
        SetRagdoll(true);

        Rigidbody closest = FindClosestBody(hitPoint);
        if (closest != null)
        {
            closest.AddForceAtPosition(hitForce, hitPoint, ForceMode.Impulse);
        }
    }

    private void SetRagdoll(bool enabled)
    {
        if (animator != null)
            animator.enabled = !enabled;

        if (navAgent != null)
            navAgent.enabled = !enabled;

        if (standingCollider != null)
            standingCollider.enabled = !enabled;

        foreach (Rigidbody body in ragdollBodies)
        {
            body.isKinematic = !enabled;
            body.detectCollisions = enabled;
        }

        foreach (Collider col in ragdollColliders)
        {
            col.enabled = enabled;
        }
    }

    private Rigidbody FindClosestBody(Vector3 point)
    {
        Rigidbody best = null;
        float bestDistance = float.PositiveInfinity;

        foreach (Rigidbody body in ragdollBodies)
        {
            float distance = Vector3.SqrMagnitude(body.worldCenterOfMass - point);
            if (distance < bestDistance)
            {
                bestDistance = distance;
                best = body;
            }
        }

        return best;
    }
}
为什么关 Animator？

Animator 会继续写骨骼 Transform，物理也想移动骨骼，两个系统会抢控制权，表现为抖动或奇怪拉扯。

为什么关 NavMeshAgent？

AI 导航还在移动角色根物体，尸体会被继续拖着走。死亡后通常要禁用 agent。

为什么关站立胶囊体？

角色倒地后，原来的大 CapsuleCollider 还竖着会挡玩家，也会让尸体碰撞很怪。

Debug

常见问题排查

尸体悬空

通常是 Collider 太大。缩小手臂、腿、头的 Radius 或 Size，让 collider 更接近 mesh。倒地时接触地面的是 collider，不是模型表面。

角色死亡后疯狂抖动

检查 Animator 是否关闭、相邻骨骼的 Enable Collision 是否关闭、刚体质量差异是否太大，以及 joint 限制角是否过小。

手臂或腿弯曲方向不对

检查 Character Joint 的 Axis。Axis 决定 Low/High Twist 的方向，自动生成后经常需要对前臂或小腿手动修正。

尸体被 AI 继续拖着走

死亡时关闭 NavMeshAgent，也要停止状态机继续设置 destination、速度或根运动。

骨骼像被拉长

可能是关节约束无法满足。避免出生在墙里，必要时打开 Enable Projection，并提高 Physics 的 solver iterations。

性能变差

不要给每个小骨头都加复杂物理。只给头、胸、骨盆、上臂、前臂、大腿、小腿等关键骨骼加 collider 和 rigidbody。
