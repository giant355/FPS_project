一句话：Character Joint 连接两个 Rigidbody，并限制它们之间能怎么旋转。

Interactive

Character Joint 就是“有限制的骨骼关节”

在 ragdoll 里，前臂、上臂、大腿、小腿、头、脊柱都还是 Rigidbody。Character Joint 负责把它们连起来，并用 Twist / Swing 限制旋转范围。

Mental Model

先别背参数，先记住四件事

Connected Body 是“连谁”

前臂 joint 的 Connected Body 通常是上臂的 Rigidbody。为空时，joint 连接到世界空间。

Anchor 是“从哪里转”

肘关节的 Anchor 应该在肘部，膝关节的 Anchor 应该在膝盖，不应该在骨头中间。

Axis 是“主旋转轴”

Low Twist 和 High Twist 都围绕 Axis 限制旋转。Axis 方向错，关节限制也会错。

Swing 是“其他方向摆动”

Swing 1 和 Swing 2 控制侧向摆动，通常比 twist 更小，避免 ragdoll 变得太松。

你截图里的参数怎么读

  • 1Connected Body = None：这个 joint 当前没有连接到另一个 Rigidbody，所以会连到世界空间。
  • 2Anchor = X 0, Y 1, Z 0：关节旋转点在当前物体本地坐标的上方。
  • 3Axis = X 1, Y 0, Z 0：主扭转轴是当前物体的本地 X 轴。
  • 4Auto Configure Connected Anchor 已勾选：Unity 自动计算连接锚点。
  • 5High Twist Limit = 70：围绕主轴正方向最多转 70 度。

Character Joint 和 Hinge Joint 的区别

Hinge Joint 更像门轴，主要绕一个轴转。Character Joint 更像人体关节，可以给主轴设置不对称限制，也能限制另外两个摆动方向，所以更适合 ragdoll。

  • A肘关节：向前弯很多，向后反折很少。
  • B膝关节：主要向后弯，反向弯曲应很小。
  • C头颈：各方向都要限制，否则头会转得很离谱。

Inspector Reference

Inspector 参数速查表

下面按你截图里的顺序解释。调 ragdoll 时，优先看 `Connected Body`、`Anchor`、`Axis`、`Low/High Twist Limit`、`Swing 1/2 Limit`。

参数 意思 怎么调
Edit Angular Limits 在 Scene 视图里显示可拖动的角度限制 gizmo。 用来可视化调限制，比纯填数字直观。
Connected Body 连接到哪个 `Rigidbody`。如果不填，就是连接到世界。 ragdoll 骨骼一般都要连到父级或相邻骨骼。
Connected Articulation Body 连接到 `ArticulationBody`，偏机器人/机械臂系统。 普通角色 ragdoll 基本不用，保持 None。
Anchor 当前物体本地空间里的关节位置。 必须放在真实关节点附近：肘、膝、肩、脖子。
Axis 主扭转轴，Scene gizmo 里通常用橙色显示。 Low/High Twist 都围绕它。方向错时，先改这个。
Auto Configure Connected Anchor 让 Unity 自动算连接物体上的锚点。 一般先勾选。自动点不准时再关闭并手动调。
Connected Anchor 连接物体本地空间里的对应锚点。 手动模式下才重点调，通常让它和 Anchor 对齐到同一个世界位置。
Swing Axis 摆动轴，Scene gizmo 里通常用绿色显示。 控制 Swing 1 方向；Swing 2 是与 Axis、Swing Axis 垂直的第三轴。
Twist Limit Spring 到达 twist 限制时是否有弹性。 ragdoll 通常先设 0，避免像橡皮筋一样弹。
Low Twist Limit 主轴负方向的旋转下限。 肘/膝常用小负值，例如 -5 到 -15。
High Twist Limit 主轴正方向的旋转上限。 截图里的 70 表示正方向最多 70 度。
Swing Limit Spring 到达 swing 限制时是否有弹性。 新手先保持 0；需要更软的物理效果再调。
Swing 1 Limit 围绕 Swing Axis 的对称摆动角度。 比如 30 表示 -30 到 +30。小于 5 度不如直接设 0。
Swing 2 Limit 第三个轴的对称摆动角度。 用于控制另一侧方向的摆动，避免四肢乱甩。
Enable Projection 关节被拉出约束时，强制拉回。 尸体拉伸、散架时再开。它稳定，但可能有突然修正感。
Projection Distance / Angle 超出多少距离或角度后开始 Projection 修正。 默认先不动；拉伸严重时再慢慢收紧。
Break Force / Break Torque 超过多少力或扭矩后 joint 断开。 角色 ragdoll 一般保持 Infinity，否则 mesh 会被拉得很怪。
Enable Collision 连接的两个刚体是否互相碰撞。 相邻骨骼通常关闭，否则轻微重叠会互相弹开。
Enable Preprocessing Unity 对 joint 做预处理，帮助求解。 默认开。遇到被墙体挤压导致的分离/乱跳时，可以试着关。
Mass Scale / Connected Mass Scale 只影响 joint 求解时的有效质量。 两个连接刚体质量差太多会抖，尽量让有效质量接近。

Practical Values

常用关节设置思路

这些不是固定答案，因为模型朝向和骨骼轴可能不同。它们是调参方向：先确认 Axis，再根据人体关节给合理角度。

C#

用代码设置 Character Joint 的限制

实际项目里大多数参数会在 Inspector 里调。代码适合做批量修正或运行时切换。

CharacterJointSetup.cs 示例：设置 twist / swing limit 
using UnityEngine;

public class CharacterJointSetup : MonoBehaviour
{
    [SerializeField] private CharacterJoint joint;
    [SerializeField] private Rigidbody connectedBody;

    private void ResetJoint()
    {
        joint.connectedBody = connectedBody;

        // Anchor 是当前骨骼本地空间里的关节点。
        joint.anchor = new Vector3(0f, 1f, 0f);

        // Axis 是主扭转轴。这里使用本地 X 轴。
        joint.axis = Vector3.right;
        joint.swingAxis = Vector3.up;

        SoftJointLimit low = joint.lowTwistLimit;
        low.limit = -10f;
        low.bounciness = 0f;
        joint.lowTwistLimit = low;

        SoftJointLimit high = joint.highTwistLimit;
        high.limit = 90f;
        high.bounciness = 0f;
        joint.highTwistLimit = high;

        SoftJointLimit swing1 = joint.swing1Limit;
        swing1.limit = 25f;
        joint.swing1Limit = swing1;

        SoftJointLimit swing2 = joint.swing2Limit;
        swing2.limit = 20f;
        joint.swing2Limit = swing2;

        joint.enableCollision = false;
        joint.enableProjection = true;
    }
}

Debug

调 Character Joint 时最常见的问题

关节弯曲方向反了

先看 Axis。Low/High Twist 都围绕 Axis，Axis 朝向错了，限制会像装反。打开 Scene gizmo 看橙色箭头最直观。

尸体一直抖

检查相邻骨骼是否互相碰撞、质量差距是否过大、角度限制是否太小。Unity 建议小角度限制如果只有几度，直接锁到 0 通常更稳定。

骨骼被拉长或分离

可能是 joint solver 满足不了约束，例如 ragdoll 出生在墙里或受力太大。可以开启 Projection，并提高 Physics 的 Default Solver Iterations。

Enable Collision 要不要开

相邻骨骼通常不要开。手臂 collider 和身体 collider 有轻微重叠很常见，打开后会互相推开，导致抖动。
